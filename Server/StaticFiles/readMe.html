4. What was hard to do?

Hagar - Key Development Challenges

Database Design and Integration: Creating a well-structured database with tables for users, followers, likes, comments, posts, and other entities is a crucial foundation. Integrating this database effectively into the application can be challenging, ensuring that data is stored and retrieved accurately.

Server Architecture: Designing a robust server architecture with a clear separation of concerns into app.js, routers, and controllers is essential for scalability and maintainability. Coordinating these components to handle various endpoints and routes can be complex.

Frontend Development: Developing individual HTML, CSS, and JavaScript files for each page of the app can become unwieldy as the project grows. Ensuring consistency in design and functionality across all pages is a challenge.

Amit - Key Development Challenges

Throughout the development process, we encountered several significant challenges:

Admin's page - enable/disable each of the additional features/pages was a difficult task.
Server's flow management was very challenging to handle.
Pages design has taken a long time using many files for each page such as .js, .html, .css.
Login Authentication has proved to be troublesome, as I encountered many problems with hashing the password along the way.
User logged in verification - authenticating which user is logged in has taken many hours of trial and error.
Making sure that Like and Dislike buttons are not pushed simultaneously by the same user, and remembered in the next page refresh.

6. Specify all the different (server) routes your app supports

Our app supports the following server routes:

  • POST /insert: inserts an entry into a table
  • POST /insertLikeIfNotExists: inserts an entry into a table if it doesnt already exist
  • POST /get_table_data: gets all table data
  • POST /get_table_data_id: gets all table data according to id
  • POST /add_full_name: adds full_name to a json element
  • POST /getUsersNotFollow: gets all users that you do not follow
  • POST /get_table_data_id_2_values: gets all table data according to 2 id values
  • POST /explore: gets all explore posts
  • POST /feed: gets all feed posts
  • POST /modify_feature: modifies feature
  • POST /remove_by_id: remove entry in database according to id
  • POST /remove_by_2_ids: remove entry in database according to 2 id values
  • POST /getUserId: gets user_id of the currently logged in account
  • POST /getActivity: gets all activities of activity_type
  • POST /addActivity: adds activity to ActivityLog
  • POST /encryptPass: encrypts password
  • POST /getEncryptedPass: gets the encrypted password
